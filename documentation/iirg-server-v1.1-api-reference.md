# iIRS Attack Graph Generator (iIRG) Server RestAPI Call Reference

**Version 1.1: 12/Oct/2019**

**Changelog**

| Version | Date        | Description |
| :-----: | :---------: | ----------- |
| 1.0 | 01/Oct/2019 | Document creation. |
| 1.1 | 12/Oct/2019 | Changes to [call #8](#8-get-the-attack-graph-as-generated-by-mulval) (GET: /attack_graph): 1) Added the vertex to topology/network hosts association tables structure. 2) Removed the `related_host` structure from all vertices. |

## List of Endpoints

**The API of the Attack Graph Engine Server is available at:**\
`http://127.0.0.1:10000/ag-engine-server/rest` for calls 1-2\
`http://127.0.0.1:10000/ag-engine-server/rest/json` for calls 3-25

**Cookie management:**\
*Calls 1, 2* and *20-25* don't require cookie management.\
*Calls 3, 4* (initialize calls) generate a cookie to identify and track each session.\
*Calls 5-19* require the use of the cookie generated by the initialize calls (3, 4).\

**General notes:**\
The numbering of the remediation options and remediation actions for calls *15-19* starts at 0.

**Status field explanation:**\
:heavy_check_mark: = Completed\
:heavy_minus_sign: = In Development / Unfinished

| #  | Request Method | Endpoint | Description | Status |
| :-: | -------------: | -------- | ----------- | :----: |
| 1  | GET  | /version | Version of the Attack Graph Engine Server in plain-text format. | :heavy_check_mark: |
| 2  | GET  | /version/detailed | Same as above, in JSON format. | :heavy_check_mark: |
| 3  | GET  | /initialize | Initialize using on-disk data. | :heavy_check_mark: |
| 4  | POST | /initialize | Initialize using an already-generated FIWARE XML topology. | :heavy_check_mark: |
| 5  | GET  | /topology | Get the provided/generated FIWARE XML topology. | :heavy_check_mark: |
| 6  | GET  | /host/list | Get the list of hosts and their security requirements. | :heavy_check_mark: |
| 7  | POST | /host/list | Set the list of hosts and their security requirements. | :heavy_check_mark: |
| 8  | GET  | /attack_graph | Get the attack graph (as generated by MulVAL). | :heavy_check_mark: |
| 9  | GET  | /attack_graph/score | Get the attack graph score. | :heavy_check_mark: |
| 10 | GET  | /attack_graph/topological | Get the topological form of the attack graph. | :heavy_check_mark: |
| 11 | POST | /attack_graph/block_nodes | Receives node IDs to block from the iIRS and sends the required firewall rules to the A16 module. | :heavy_minus_sign: |
| 12 | GET  | /attack_graph/remediations/all | Get all available remediations of every attack path and the affected nodes of each remediation. | :heavy_minus_sign: |
| 13 | GET  | /attack_path/list | Get all attack paths. | :heavy_check_mark: |
| 14 | GET  | /attack_path/number | Get the number of attack paths. | :heavy_check_mark: |
| 15 | GET  | /attack_path/{id} | Get the specified attack path. | :heavy_check_mark: |
| 16 | GET  | /attack_path/{id}/topological | Same as above but in its topological form. | :heavy_check_mark: |
| 17 | GET  | /attack_path/{id}/remediations | Get the remediations for the specified attack path. | :heavy_check_mark: |
| 18 | GET  | /attack_path/{id}/remediation/{id_remediation} | Simulate the specified remediation on the specified attack path and compute the new attack graph. | :heavy_check_mark: |
| 19 | GET  | /attack_path/{id}/remediation/{id_remediation}/validate  | Validate that the specified remediation has been applied. | :heavy_check_mark: |
| 20 | POST | /topology/net-ip | Set the networks (in CIDR format) to be considered by /initialize with data on disk. | :heavy_check_mark: |
| 21 | POST | /topology/vuln-scan-report | Upload the vulnerability scan report results, to be used by /initialize with data on disk. | :heavy_minus_sign: |
| 22 | POST | /topology/hosts-interfaces | Upload the hosts-interfaces.csv file contents, to be used by /initialize with data on disk. | :heavy_check_mark: |
| 23 | POST | /topology/vlans | Upload the vlans.csv file contents, to be used by /initialize with data on disk. | :heavy_check_mark: |
| 24 | POST | /topology/flow-matrix | Upload the flow-matrix.csv file contents, to be used by /initialize with data on disk. | :heavy_check_mark: |
| 25 | POST | /topology/routing | Upload the routing.csv file contents, to be used by /initialize with data on disk. | :heavy_check_mark: |
